⚙️ Setup required — PO Deliveries sheet

This tab reads from a PO_Deliveries sheet tab in the QueryData spreadsheet, populated by an Apps Script sync. To set it up:

  1. Open the QueryData sheet → Extensions → Apps Script
  2. Add a second sync function (e.g. syncDeliveries()) using the SQL below
  3. Run it once manually to create the PO_Deliveries tab and note its sheet GID
  4. Get the CSV export URL:
    https://docs.google.com/spreadsheets/d/1LnMn_BU0DIHSOihsTVS0kl0bWITWlxayvdgfMtYoKyQ/export?format=csv&gid=YOUR_GID
  5. Paste the URL above and click Load Deliveries
  6. Optionally set a daily trigger on syncDeliveries() to keep it fresh
▶ Show SQL for Apps Script 
DECLARE @TODAY DATETIME = GETDATE();
DECLARE @PAST  DATETIME = DATEADD(DAY,-90,GETDATE());
SELECT
  p.ExternalInvoiceNumber PONumber, s.SupplierName, CAST(' '+si.ItemNumber AS NVARCHAR(51)) SKU,
  si.ItemTitle, pi.Quantity OrderedQty, pi.Delivered ReceivedQty,
  pi.Quantity-pi.Delivered OutstandingQty,
  CAST(pi.Cost AS DECIMAL(10,2)) UnitCost,
  CAST((pi.Quantity-pi.Delivered)*pi.Cost AS DECIMAL(10,2)) OutstandingValue,
  CONVERT(VARCHAR(10),p.DateOfPurchase,103) PODate,
  CONVERT(VARCHAR(10),p.QuotedDeliveryDate,103) ExpectedDelivery,
  CONVERT(VARCHAR(10),p.DateOfDelivery,103) ActualDelivery,
  p.Status POStatus,
  DATEDIFF(DAY,p.QuotedDeliveryDate,@TODAY) DaysOverdue
FROM Purchase p
JOIN PurchaseItem pi ON pi.fkPurchasId=p.pkPurchaseID
JOIN StockItem si ON si.pkStockItemID=pi.fkStockItemId
JOIN Supplier s ON s.pkSupplierId=p.fkSupplierId
WHERE p.DateOfPurchase>=@PAST AND pi.Quantity>0 AND p.Status NOT IN('CANCELLED')
ORDER BY CASE p.Status WHEN 'PENDING' THEN 1 WHEN 'PLACED' THEN 2 WHEN 'PARTIAL' THEN 3 WHEN 'DELIVERED' THEN 4 ELSE 5 END, p.QuotedDeliveryDate ASC
